## 介绍

LongLive-Lib，简称 `3L`，是一个实验性的标准库。

它的目标很简单：一方面给玩家带来一些实用的优化和更好的使用体验，另一方面也给 Modder 提供更顺手的调试工具、兼容能力，以及以后继续往下扩展的基础。

`3L` 目前建立在 `BepInEx Plugin` 和 `Next` 框架之上。
理论上它可以脱离 `Next` 单独运行，但从兼容性、生态协作和实际体验来看，**还是更建议和 Next 一起使用**。

因为 `3L` 在启动时会额外加载一些运行时内容，所以**刚进入游戏时加载时间变长是正常现象**。

这个项目会一直保持开源。
如果你想看源码或者参与开发，可以在 GitHub 上搜索 `LongLive-Lib`，仓库是：
`Nanaloveyuki/LongLive-Lib`

如果你本身会写 C#，也欢迎来一起折腾。

## 当前为玩家提供的功能

1. **大量使用物品时的卡顿优化**
当你一次性使用很多丹药、妖丹或者其他可消耗物品时，`3L` 会尽量减少卡顿，让结算过程没那么容易把游戏拖慢。

如果你不喜欢这个功能，可以在 `F1` 的 Mod 配置界面中关闭：
`EnableBulkItemUseOptimization`

2. **提示信息（PopTip）合并与清理优化**
当游戏在短时间内弹出很多提示时，`3L` 会尝试把重复提示合并起来，顺便清理堆积的提示队列，并加快大量提示出现时的淡入淡出速度。

这样做主要是为了减少提示刷屏挡住视线的问题，也能避免一些提示在不该停留的时候停太久。

如果你不喜欢这个功能，可以在配置里关闭：
`EnablePopTipOptimization`

3. **战斗数值结算保护**
在某些高伤害、多段攻击，或者数值本来就比较离谱的战斗情况下，原版有时会出现目标已经进入负血，后续技能段却还在继续结算的情况。

这类情况不仅会让战斗变卡，还可能带来音效叠在一起、表现异常之类的问题。

`3L` 现在提供了一套实验性的保护逻辑，会先做一部分数值判断，尽量减少“目标已经死了，后面的伤害还在继续打”的情况。

如果你不想启用这个功能，可以在配置中关闭：
`EnableExperimentalBattleGuard`

4. **部分黑屏过场与切换动画加速**
`3L` 现在还会对一部分共享黑屏、开门过场和场景切换流程进行加速，让重复进出场景时不至于显得太拖。

如果你不喜欢这个功能，可以在配置中关闭：
`EnableFadeOptimization`

5. **主菜单诊断入口**
主菜单里会有一个比较明显的 `LongLive Lib` 入口，用来查看 Host / Bridge 状态、兼容层状态，以及部分运行时诊断信息。

对普通玩家来说，这个入口也挺有用，因为它可以帮你确认：这个 Mod 到底有没有装对、有没有正常加载。

## 当前为 Modder 提供的功能

1. **战斗 Tracer**
可以记录比较详细的战斗日志，比如用了什么技能、结算是怎么走的、关键事件是怎么触发的，方便排查复杂战斗问题。

2. **地图 Tracer**
可以记录当前地图状态、位置、节点和相关参数，方便分析场景切换和地图运行过程。

3. **地图快照自动导出**
这是未来自定义地图功能的前置调试工具。
它可以帮助开发者观察地图结构、整理节点信息，也为后续地图扩展提前打基础。

4. **Native Probe**
这个功能是为以后接入 `Rust` 原生计算核心准备的，目前主要用于验证 DLL 是否能正确加载，以及调用链路是否打通。

5. **图鉴拼音搜索增强**
`3L` 现在已经为图鉴搜索这条路径提供了拼音搜索支持，也算是后续中文搜索扩展的一个先行尝试。

6. **Host / Bridge 诊断信息**
`3L` 现在会提供比较详细的 Host / Bridge 诊断信息，包括安装状态、兼容开关、运行能力摘要，以及一些方便排查问题的状态信息。

7. **JSON Demo 安装**
这个功能目前还只是很早期的实验内容。
它离真正可用的 JSON Mod 机制还有距离，不过已经算是在为以后铺路了。

## 注意事项

- 这个 Mod 还在开发中，功能和接口后面都可能继续调整
- 启动加载时间比平时稍长是正常现象
- 更推荐和 `Next` 一起使用，这样兼容性会更好一些
- 有些功能本来就更偏向优化和调试，不一定完全符合每个人的口味，不喜欢的话可以按需关闭
